## 修复的 Bug

### 14.1 Target weight 列类型不匹配（Critical）

**根因**: `target_schema()` 中 weight 字段使用 `.float("weight")`（映射为 FLOAT8），但 DB schema 定义为 `INTEGER`。

**修复**: 改为 `.integer("weight")`。

**影响文件**:
- `crates/kong-db/src/dao/postgres.rs` — 第 1160 行

**Commit**: `243b2a6`

### 14.2 Docker 容器 workspace migration 冲突导致无限重启（Critical）

**根因**: `001_add_workspaces.sql` 中 INSERT 使用 `ON CONFLICT (id) DO NOTHING`，仅处理 id 列冲突。当已有 Kong DB 中 `default` workspace 使用不同 UUID 时，name 唯一约束冲突未被捕获，导致 migration 失败 → 进程崩溃 → restart:always 循环重启。

**修复**: 改为 `ON CONFLICT DO NOTHING`（无条件忽略所有唯一约束冲突）。

**影响文件**:
- `crates/kong-db/migrations/core/001_add_workspaces.sql` — 第 21 行

**Commit**: `7ce46f8`
