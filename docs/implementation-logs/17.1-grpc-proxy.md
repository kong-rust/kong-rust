# 17.1 完整 gRPC 代理支持

## 概述

实现 Kong-Rust 的完整 gRPC 代理支持，包括 gRPC 请求检测、gRPC 格式错误响应、trailer 保留和流式传输支持。

## 修改文件

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `crates/kong-proxy/src/grpc.rs` | 新建 | gRPC 辅助模块：状态码映射、请求检测、错误响应 |
| `crates/kong-proxy/src/lib.rs` | 修改 | 集成 gRPC 支持到代理引擎各阶段 |
| `crates/kong-proxy/tests/proxy_e2e.rs` | 修改 | 新增 10 个 gRPC 测试用例 |

## 实现细节

### 新建 grpc.rs 模块

- **GrpcStatus 枚举**：定义 gRPC 状态码（与 gRPC 规范一致的数值）
- **http_status_to_grpc()**：HTTP 状态码 → gRPC 状态码映射（Kong 兼容）
  - 404 → UNIMPLEMENTED (12)
  - 503 → UNAVAILABLE (14)
  - 500 → INTERNAL (13)
  - 401 → UNAUTHENTICATED (16)
  - 403 → PERMISSION_DENIED (7)
  - 429 → RESOURCE_EXHAUSTED (8)
- **is_grpc_request()**：通过 content-type: application/grpc 检测 gRPC 请求
- **send_grpc_error()**：发送 gRPC Trailers-Only 错误响应（HTTP 200 + grpc-status + grpc-message headers）

### lib.rs 代理引擎集成

1. **KongCtx 新增 `is_grpc_request` 字段**：在 request_filter 阶段检测并设置
2. **send_error_or_grpc() 方法**：统一错误响应入口，自动选择 JSON 或 gRPC 格式
3. **所有框架级错误点升级**：404（无路由）、400（请求体错误）、500（插件错误）、503（无服务/服务禁用）
4. **trailer 头保留**：gRPC 请求不剥离 upstream_request_filter 和 upstream_response_filter 中的 trailer 逐跳头
5. **Body buffering 禁用**：gRPC 请求强制 request_body_filter 和 response_body_filter 为流式模式

### 已有基础设施（无需修改）

- Server 端 h2c 支持（main.rs: `server_opts.h2c = true`）
- 上游 h2c 先验知识（`peer.options.set_http_version(2, 2)` for plaintext gRPC）
- 上游 H2H1 ALPN（TLS gRPC 自动 HTTP/2 协商）
- 路由匹配透明映射（grpc→http, grpcs→https）
- Admin API gRPC 路由约束验证（禁止 methods、禁止 strip_path）

## 测试

新增 10 个测试（全部通过）+ 2 个 grpc.rs 内部单元测试：

- `test_grpc_route_matching_by_host` — 按 host 匹配 gRPC 路由
- `test_grpc_route_matching_by_path` — 按 path 匹配 gRPC 路由
- `test_grpc_route_matching_https_scheme` — grpcs 匹配 HTTPS 请求
- `test_grpc_route_does_not_match_wrong_host` — 错误 host 不匹配
- `test_grpc_and_http_routes_coexist` — gRPC 和 HTTP 路由共存
- `test_grpc_route_strip_path_forced_false` — strip_path 约束
- `test_grpc_upstream_resolution` — 上游协议检测（Grpc/Grpcs → TLS/non-TLS）
- `test_grpc_status_code_mapping` — HTTP→gRPC 状态码映射验证
- `test_grpc_route_with_sni` — SNI 匹配 gRPC 路由
- `test_http_to_grpc_status_mapping` — 完整映射覆盖
- `test_grpc_status_code_values` — gRPC 状态码数值正确性
