- **Status:** Completed

## Summary
为 Kong-Rust 项目实现了完整的 HTTP/2 支持，包含客户端 (Downstream) 到代理的连接支持，以及代理到上游 (Upstream) 的连接支持。通过配置 Pingora 底层的 TLS 设置和 ALPN 协商策略，满足了 Kong 原始的协议支持行为。

## Changes Made
- **下游 HTTP/2 支持**：修改了 `crates/kong-server/src/main.rs`。针对配置为带有 `http2` 标记的 TLS 端口，改用 `proxy_service.add_tls_with_settings` 并通过 `tls_settings.enable_h2()` 在 TLS 握手中开启 `h2` 协议协商，从而允许客户端通过 HTTP/2 进行连接。
- **上游 HTTP/2 支持**：修改了 `crates/kong-proxy/src/lib.rs` 中创建 `HttpPeer` 的逻辑。如果服务配置中启用了 `upstream_tls`，则将底层的 `peer.options.alpn` 设置为 `pingora_core::protocols::tls::ALPN::H2H1`，使得代理在连接上游服务端时优先协商使用 HTTP/2。
